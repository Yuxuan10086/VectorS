来源：`PD42S1步进电机闭环驱动器用户手册_V1.2.pdf`、`正点原子步进电机驱动器控制协议V1.2.xlsx`。

## 上下行帧结构（串口 TTL / RS485 / CAN 载荷一致）

| 字段 | 下行（上位机→驱动板） | 上行（驱动板→上位机） |
|------|------------------------|------------------------|
| 帧头 | `0xC5` | `0xC5` |
| 从机地址 | `addr`（0 广播，1–255） | `addr` |
| 功能码 | `code` | `code`（通常与对应指令相同） |
| 指令数据 | 视功能码而定 | 视功能码而定；首字节多为操作结果 |
| 检验码 | `CHECKSUM` | `CHECKSUM` |
| 帧尾 | `0x5C` | `0x5C` |

**校验（8 位）**

`CHECKSUM = (0xC5 + addr + code + 指令数据所有字节) & 0xFF`

---

## 默认与常用参数

| 项目 | 默认值 / 说明 |
|------|----------------|
| 自定义协议 | 使用自定义协议前需 **关闭 Modbus**（屏幕菜单 MODBUS→失能，或指令 `0x69`） |
| 从机地址 | 出厂 **0x01** |
| 串口 TTL / RS485 | **115200**，8 数据位，1 停止位，无校验 |
| CAN | **500 kbit/s**，仅 **扩展帧**；报文 **ID** 需符合 **0x100x**（如 0x1001） |
| CAN 单帧数据 | 每条 CAN 数据段最多 **8 字节**，更长指令需拆成多帧连续发送 |
| 供电（手册产品规格） | **12–32 V**（示例常用 **24 V**） |

---

## 注意事项（时序与应答）

1. **单条指令内**：从 `0xC5` 到 `0x5C` 须 **连续发送**，字节间不宜超过 **1 个字节时间** 的间隔，否则可能接收失败。  
2. **指令与指令之间**：两条完整帧之间至少间隔 **2 ms**。  
3. **广播地址 0**、**分组广播**：从机 **不应答**。  
4. **总线上无对应从机地址**：**不应答**。  
5. 可通过 **`0x6F` 指令回响** 配置为 **不回显应答**（不需要应答时使用）。  
6. 上行帧指令数据第一字节为操作结果：**`0x01` 成功**；`0xE1`–`0xE6` 为错误码（帧长、帧头尾、校验、功能码、数据等）。
